# Contributing to Enterprise AI Blueprints

Thank you for your interest in contributing to **Enterprise AI Blueprints**.

We welcome contributions from architects, developers, cloud engineers, researchers, and AI practitioners.

---

## Ways to Contribute

You can contribute by:

- 📘 Writing architecture documentation
- 🏗 Creating Enterprise AI reference architectures
- 🤖 Adding Agentic AI patterns
- ☁️ Cloud deployment examples
- 🔒 Security best practices
- 🧪 Example implementations
- 🐛 Reporting issues
- 💡 Suggesting improvements

---

## Getting Started

1. Fork the repository.

2. Create a feature branch.

```bash
git checkout -b feature/my-blueprint
```

3. Commit your changes.

```bash
git commit -m "Add Enterprise RAG blueprint"
```

4. Push your branch.

```bash
git push origin feature/my-blueprint
```

5. Open a Pull Request.

---

## Documentation Standards

Please follow these guidelines:

- Use Markdown
- Include architecture diagrams
- Explain design decisions
- Provide deployment recommendations
- Include references where appropriate

---

## Code Standards

- Keep examples simple.
- Follow cloud-native best practices.
- Write reusable documentation.
- Keep architectures vendor-neutral whenever possible.
